## les modèles composites
Un B-26 n'est pas simplement : B26.obj + B26.mtl

On a plutôt :
```
B-26
│
├── B-26.obj
├── B-26_Dorsal Turret.obj
├── B-26_Flightdeck.obj
├── B-26_Fwd Bomb Bay_Closed.obj
├── B-26_Fwd Bomb Bay_Open.obj
├── B-26_GearDown.obj
├── B-26_GearUp.obj
├── B-26_PropBlur.obj
├── B-26_PropPort.obj
├── B-26_PropStbd.obj
├── Bomb.obj
└── Ladder.obj
```

extension envisagée : ModelDescriptor
```js
{
    id   : 'b26',
    name : 'Martin B-26 Marauder',

    classification : {
        category : 'avions',
        type     : 'aircraft',
    },

    resource : {
        format : 'composite',
        path   : null,
    },

    parts : [
        'b26-main',
        'b26-dorsal-turret',
        'b26-flightdeck',
        'b26-gear-down',
    ],
}
```

## Relation entre model

### Relation de composition
Un B26 a besoin du model complémentaire Dorsal Turret

Ceci permet de gérer 
- des versions : B-26-A , B-26-B
- les animations des enfants

``` 
B-26
└── Dorsal Turret
```

### Relation d'association
Un B-26 "peut" avoir un enfant Tallboy, Tallboy peut etre ajouté a un autre model Lancaster
```
B-26
└── compatible avec → Tallboy
Lancaster
└── compatible avec → Tallboy
```


```
ModelDescriptor
```

## textures

### textures géoréférencées

shom11.png est intéressant car ce n'est pas seulement une texture visuelle.

Elle peut avoir :
```
Texture
│
├── image
├── projection
├── bounding box géographique
├── résolution
└── coordonnées
```
Par exemple :
```js
{
    id   : 'shom11',
    type : 'texture',

    resource : {
        path   : '/assets/img/3js/textures/shom11.png',
    },

    geo : {
        enabled : true,
        crs     : '...',
        bounds  : {
            minLat : ...,
            maxLat : ...,
            minLng : ...,
            maxLng : ...,
        },
    },
}
```

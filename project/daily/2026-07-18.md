Recommandation sur les dimensions

Solution hybride recommandée :
- On garde le transform.targetSize (très utile pour normaliser à l’import)
- On ajoute un bloc dimensions réelles (optionnel) dans le ModelDescriptor

Règles de gestion recommandées

| Usage | Source recommandée | Commentaire |
| --- | --- | --- |
| Affichage dans l’Inspector | realWorld | Dimensions réelles (utilisateur) |
| Normalisation à l’import | transform.targetSize | Très utile |
| Calculs physiques / collisions | computed.boundingBox | Après scale appliqué |
| Comparaison d’échelle | realWorld + computed.boundingBox | Pour vérifier cohérence |

Logique :À l’import → on peut calculer un scale automatique pour correspondre à targetSize.
L’utilisateur peut overrides manuellement le scale dans l’Inspector.
Les realWorld dimensions restent toujours la référence "vraie".
